## Usage Guidelines

### Primary Actions & CTAs
Use `--accent` (#BF3143) for:
- Primary buttons
- Important links
- Call-to-action elements

### Positive Status & Growth
Use sage greens for:
- "Live" badges
- Success states
- Growth indicators
- Completion states

### Text Hierarchy
- **Headlines**: `--text` (#1e293b)
- **Body/Secondary**: `--muted` (#64748b)
- **Muted/Tertiary**: Reduce opacity of `--text`

### Surfaces & Cards
- **Background**: `--card` (#ffffff)
- **Hover**: `--soft` (#f0f4f3)
- **Border**: `--border` (#e2e8f0)

### Illustrations
Stick to the plant palette (#a6c5a3, #cbdfa5, #7596b3, #324252) for:
- Decorative SVGs
- Icons
- Visual consistency

### Popups, Modals & Alerts
All popups, modals, dialogs, toasts, and alert messages **must** follow this palette:
- **Background**: `--card` (#ffffff) with `--border` (#e2e8f0) border
- **Overlay/Backdrop**: `rgba(30, 41, 59, 0.5)` (semi-transparent `--text`)
- **Border radius**: `--radius` (16px)
- **Title text**: `--text` (#1e293b)
- **Body text**: `--muted` (#64748b)
- **Primary action button**: `--accent` (#BF3143) background with white text
- **Secondary/Cancel button**: `--soft` (#f0f4f3) background with `--text` color
- **Success alerts**: Use the Success Pattern (green) colors
- **Error alerts**: Use `--accent` (#BF3143) for error states
- **Warning alerts**: Use Weekend Pill colors (#F7F5FA background, #6E6282 text)
- **Info alerts**: Use `--soft` (#f0f4f3) background with `--text` color

## Accessibility Notes

- **Text on white background**: Use `--text` (#1e293b) for AAA contrast
- **Muted text**: `--muted` (#64748b) provides AA contrast on white
- **Accent red**: Ensure sufficient contrast when used with text
- **Green pills**: Dark text (#324252) on light background (#e8f4ed) ensures readability

## Iconography Guidelines

### No Emojis Policy
**Do not use emojis** in TinyWins applications. Emojis appear unprofessional and inconsistent across platforms.

### Icon Style
Use **simple, elegant SVG icons** that align with the TinyWins aesthetic:

- **Stroke-based**: 1.5px - 2px stroke weight
- **Rounded caps**: `stroke-linecap="round"` and `stroke-linejoin="round"`
- **Minimal detail**: Single-path or simple multi-path designs
- **Consistent sizing**: 20px, 24px, or 32px viewBox
- **Color**: Use `currentColor` to inherit from parent text color

### Recommended Icon Sources
1. **Lucide Icons** (lucide.dev) - Clean, consistent, MIT licensed
2. **Heroicons** (heroicons.com) - Tailwind-style, outline and solid variants
3. **Feather Icons** (feathericons.com) - Minimalist, stroke-based

### Icon Color Usage
| Context | Color | Usage |
|---------|-------|-------|
| Default | `--text` (#1e293b) | Standard UI icons |
| Muted | `--muted` (#64748b) | Secondary/inactive icons |
| Primary | `--accent` (#BF3143) | Active states, CTAs |
| Success | `--greenText` (#324252) | Positive actions, confirmations |
| On dark bg | `#ffffff` | Icons on accent-colored buttons |
